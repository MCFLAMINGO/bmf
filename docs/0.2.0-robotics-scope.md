# BMF 0.2.0 — Robotics Validators (scoping v2)

Author: wildwallet-agent + Erik

## Why this ships now

BMF 0.1.0 solved "is this GLB actually a rigged humanoid, and can I trust it
to run wave/breathe/crouch?" That got us Arthur, and got the SDK on npm.

The next revenue surface isn't more character files — it's **robotics**. In
mid-2026 the labs shipping and selling foundation-model policies for physical
robots are:

- **Physical Intelligence** — π0, π0.5, π0.6, π0.7 (~$5.6B valuation).
  Architecture: VLM backbone (PaLI-based ~3B params) + FAST action tokenizer
  (DCT + codebook). 50-step chunks @ 50Hz. Multi-camera (primary + L wrist +
  R wrist) + proprio. Ships via **LeRobot** and their own **openpi** repo.
- **Skild AI** — Skild Brain, $14B valuation. **Omni-bodied**: single model
  for humanoids/quadrupeds/arms/mobile without form-factor retraining.
  Layered: high-level manipulation/navigation + low-level joint/torque head.
  Trained on 100k procedurally-generated robot bodies.
- **Figure AI** — Figure 03 + **Helix 02** VLA. Dual-system: VLM reasoning
  + fast visuomotor policy. Whole-body control (upper + lower). BMW
  Spartanburg deployment. 3g fingertip force sensitivity.
- **Nvidia** — GR00T / GR00T N2. Flow-matching + diffusion action heads.
  Bundled with Isaac Sim.
- **Tesla Optimus** — V3 targeting 37 joints, 22-DoF hands, AI5 chip. Ships
  no public SDK; verifiable capabilities only via URDF-equivalent leaks.
  Runs on Grok for voice.
- **Chinese frontier** — **Unitree** (G1, H1 humanoids; Go2 quadruped) ships
  URDF + their own G1_SDK. **UBTECH** Walker S2. **Fourier** GR-2.
  **Agibot / Zhiyuan** A2 humanoid. **XPeng** Iron humanoid. Each ships
  slightly different manifest formats.
- **Toolchain** — **LeRobot** (HuggingFace) has become de-facto Python
  runtime; openpi / openVLA / RT-2 formats coexist.

Their integration question is identical to Nike's shoe question: "does this
artifact ACTUALLY have the capability it claims, and can I verify that in
<100ms in production before I let it move a $30k arm?"

BMF 0.1.0 says "we can't tell you, this is declared-only."
BMF 0.2.0 says "signed answer in <50ms, no ML inference required."

That's the licensing hook. A lab can drop us in as the safety/verification
layer between their policy server and the robot. Every deploy of every policy
they ship reads a BMF manifest. That is the market.

## Erik's five, resolved

1. **Kind names — pick the least committal.** Not `urdf`/`mjcf`/`policy`,
   which lock us to one format. Use:
     - `kinematics` — any robot-body descriptor (URDF, MJCF, Unitree G1_SDK,
       Agibot robot.yaml, USD/OpenUSD, custom XML). Parser dispatched on
       internal magic. New formats are additive.
     - `policy` — any control-policy bundle (LeRobot, openpi, OpenVLA, RT-2,
       Helix-format, Skild Brain manifest, custom). Same dispatch pattern.
     - `trajectory` — any recorded demonstration set (episodes + metadata).
   The **capability namespace** stays vendor-neutral (`kin.arm`,
   `policy.obs.rgb`, `skill.pick_place`). Vendors that ship a different
   format just implement a parser adapter; the capabilities they can claim
   are the same. This is the whole point.

2. **Frontier survey — done.** New capabilities added below to cover
   patterns we missed in v1: chunked actions, VLA architecture family,
   language conditioning, wrist cameras, tactile, cross-embodiment claims,
   locomotion vs whole-body skills.

3. **Signed refusal audit log — adopted.** New capability
   `safety.refusal.audit`. When a runtime encounters `safety.simonly` on
   hardware and refuses, it MUST emit an Ed25519-signed refusal record
   (timestamp, policy hash, runtime pubkey, hardware fingerprint). Records
   are verifiable offline. That's the compliance story labs pay for.

4. **Python + Node — reference in Rust, if fast; native fallback if not.**
   Robotics ships Python (LeRobot, openpi are Python). Node is where
   dashboards and CI live. Building both from Rust core via PyO3 + napi-rs
   is the "one source of truth" solution — but only if it saves time
   downstream, not costs it. **Rule I'm setting:** try Rust core for 2
   days. If parsers aren't all done by end of day 2, cut over to native
   TypeScript + native Python (each ~600 lines of clean parser code) and
   accept the drift risk in exchange for shipping on time.

5. **Fixtures — dogfood via own gateway from 0.2.0.** GitHub Releases as
   origin blob, our gateway as verified CDN. If we can't serve our own
   fixtures reliably at 0.2, we can't sell verification to a lab that
   needs to serve theirs.

## Non-goals (unchanged)

- No policy execution. BMF verifies capabilities; it does not run the policy.
- No motion planning. `safety.workspace.limits` reports declared limits.
- No physics sim.
- No ROS bindings in 0.2.0.

## Bundle formats

### `kinematics` bundle
- Single file: `.urdf`, `.mjcf`, `.xml`, `.g1sdk.json`, or `.robot.yaml`
- Parser dispatch on magic:
  - `<robot ...>` → URDF
  - `<mujoco ...>` → MJCF
  - JSON with `{"robot_type": "unitree_g1"}` → Unitree
  - YAML with `!!agibot/robot` → Agibot
- Extension is a hint only; magic bytes win.

### `policy` bundle
- `.tar.zst` (zstd for speed) containing:
  - `manifest.json` (required, top-level, canonical schema below)
  - `weights/` (opaque; we don't touch)
  - `config.json` (framework-specific; used for fingerprint verification)

Manifest schema (canonical, all optional except `framework` and version):

```json
{
  "bmf_policy_version": "0.2.0",
  "framework": "lerobot | openpi | openvla | rt2 | groot | helix | skild | custom",
  "arch": {
    "family": "vla | diffusion | flow_matching | fast_tokenizer | autoregressive | custom",
    "backbone": "paligemma-3b | qwen3-vl-4b | custom",
    "action_head": "fast | flow_matching | diffusion | autoregressive"
  },
  "observation_space": {
    "rgb":      { "shape": [H, W, 3], "cameras": ["primary", "wrist_l", "wrist_r"] },
    "rgbd":     { "shape": [H, W, 4] },
    "proprio":  { "shape": [N], "includes": ["joint_positions", "joint_velocities", "gripper"] },
    "tactile":  { "shape": [F, C] },
    "language": { "max_tokens": 512 }
  },
  "action_space": {
    "joint":    { "shape": [N], "limits": [[lo, hi], ...] },
    "ee_pose":  { "shape": [7] },
    "gripper":  { "shape": [1], "range": [0, 1] },
    "chunk_size": 50,
    "control_hz": 50
  },
  "embodiment": {
    "targets": ["arm", "humanoid", "quadruped", "mobile"],
    "supported_robots": [
      { "name": "Franka Panda",    "robot_hash": "sha256:..." },
      { "name": "Unitree G1",      "robot_hash": "sha256:..." }
    ],
    "crossbody": true
  },
  "safety": {
    "dead_man": true,
    "sim_only": false,
    "max_ee_velocity_m_s": 0.5,
    "max_force_N": 25.0
  }
}
```

Verification is purely structural. No weight loading, no inference. Framework
fingerprint uses known files (e.g. lerobot has `config.json` with
`type: "lerobot"`).

### `trajectory` bundle
- `.tar.zst` containing:
  - `manifest.json` (required)
  - `episodes/*.npz` (numpy arrays: joint states, timestamps, images)

Manifest:

```json
{
  "bmf_trajectory_version": "0.2.0",
  "episode_count": 500,
  "duration_seconds_total": 12000,
  "robot_hash": "sha256:...",
  "task": "pick_place | teleop_replay | locomotion | whole_body | dexterous | custom",
  "success_rate": 0.94,
  "annotations": {
    "chain_of_thought": true,
    "sub_goals": true
  }
}
```

## Signing / trust

Unchanged from 0.1.0. Ed25519 over canonical manifest. New capabilities are
just more entries in the `capabilities` array. `safety.refusal.audit` is
special: it's issued by a **runtime** (not by the artifact producer), so it
uses a separate runtime-issuer key.

## Schema bump

`capabilities.registry.json` → `$id` bumped to `/schema/0.2.0/`, `version:
"0.2.0"`. All `verifiable: false, planned: "0.2.0"` flip to `verifiable:
true`. New capabilities added. Old capabilities untouched — 0.1.0 clients
still work on 0.1.0 manifests; unknown 0.2.0 capabilities are ignored per
spec §4.3.

## Gateway changes

- New kinds registered: `kinematics`, `policy`, `trajectory`
- MIME/magic sniffing:
  - `<robot>` XML → kinematics/urdf
  - `<mujoco>` XML → kinematics/mjcf
  - JSON with `robot_type` → kinematics/vendor
  - `.tar.zst` with `manifest.json.framework` → policy
  - `.tar.zst` with `manifest.json.episode_count` → trajectory
- Upload cap raised to 500MB (URDFs are small but policy bundles with
  weights can be big). Signed override for labs.
- Content-security: never execute. Refuse pickle files inside policy
  bundles (labs must use safetensors).

## CLI changes

```
bmf verify robot.urdf              → capabilities table
bmf verify policy.tar.zst          → capabilities + safety flags
bmf verify traj.tar.zst            → capabilities + skill claims

bmf check-safety policy.tar.zst --hardware
                                    → exit 1 if safety.simonly (with signed record)

bmf inspect robot.urdf --landmarks → matched landmark table (debug)
bmf inspect policy.tar.zst --arch  → arch family + backbone + action head
```
