## What shipped in Days 1-2

### Rust workspace (`bmf/core-rs/`)

Three crates:

- `bmf-core` — pure Rust. URDF + MJCF parsers, normalized IR, capability derivations.
- `bmf-py`   — PyO3 bindings. `bmf_core.parse_urdf(str) -> str` and `parse_mjcf(str) -> str`.
- `bmf-node` — napi-rs bindings. `parseUrdf(str) -> str` and `parseMjcf(str) -> str`.

FFI surface is intentionally tiny: JSON string in, JSON string out. Wrappers on
each language side do `JSON.parse` / `json.loads` once. This means we never
marshal PyDicts or napi objects per-call — big FFI-cost win, and it kept the
2-day budget realistic.

### Parsers

- **URDF** — flat `<link>` + `<joint parent/child>` list. `roxmltree`-based.
  Reference: http://wiki.ros.org/urdf/XML
- **MJCF** — nested `<body>` tree, joints child-of-body, `<freejoint/>` short
  form supported (Menagerie standard). Reference:
  https://mujoco.readthedocs.io/en/stable/XMLreference.html

Both lower to the same [`Robot`] IR (link list + joint list + `SourceFormat`
provenance). Joint kinds are: Fixed, Revolute, Continuous, Prismatic, Planar,
Floating.

## Verdict: **Rust holds. Continue Rust core path.**

Concrete signals:

1. **PyO3 + napi-rs both compiled first try** (once I fixed a naming shadow in
   bmf-py). No stalled toolchain, no undefined-behavior traps, no memory
   management surprises.
2. **Build times reasonable.** Cold Rust build ~30s; incremental <2s. Fits
   the "iterate in seconds not minutes" bar.
3. **FFI shape works.** JSON-string round-trip is boring in a good way. The
   overhead of one `json.loads` per parse is trivial relative to the parse itself.
4. **The parser survived reality.** Real Menagerie MJCF files use `<freejoint/>`,
   not `<joint type="free"/>`. Real Panda URDF has a gripper on the end of the
   arm. Real Unitree G1 has 10-DoF arm+hand chains. The morphology heuristics
   needed one iteration but converged with clear, defensible category boundaries
   (`n_short` / `n_medium` / `n_long` chain buckets).

Anti-signals we should watch for on Day 3+:

- **When we add USD/OpenUSD, `openusd-rs` is nascent.** May need to shell out
  to the C++ `usdcat` or defer to 0.2.1. Non-blocking for now — scope v3 already
  marks `kin.usd` as 0.2.1.
- **`pickle` refusal on Day 10 needs Python-side interop.** The gateway ingest
  path is TS+Node, so the pickle detection lives on the Node side (magic-byte
  sniff before we ever hand bytes to Rust). Manageable.
